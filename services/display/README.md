# Дисплей ST7789 (SPI), питание 3.3 V

Документ описывает подключение **модуля с выводами VCC, GND, DIN, CLK, CS, DC, RST, BL** к Raspberry Pi **при питании логики с 3.3 V**, как принято в `pi_remote` (переменные `.env` / `.env.example`), и кратко — **подсветку**.

**Питание:** **VCC → только 3.3 V** (пины **1** или **17** на J8). Не подавайте **5 V** на VCC, если даташит/шелкография модуля не допускает это явно.

## Схема подключения к пинам Raspberry Pi (J8)

Соответствие **выводов модуля** → **физический пин 40-pin заголовка** и **BCM** (как в коде и `raspi-config`).

| Вывод модуля | Куда на Pi        | BCM / шина   | Физ. pin J8 | Переменная / заметка        |
| ------------ | ----------------- | ------------ | ----------- | --------------------------- |
| **VCC**      | **3.3 V**         | 3V3          | **1** или **17** | Питание логики ST7789 |
| **GND**      | **GND**           | —            | **6**, 9, 14, 20, … | Общий                 |
| **DIN**      | **MOSI** (SPI0)   | **GPIO 10**  | **19**      | Линия данных SPI            |
| **CLK**      | **SCLK** (SPI0)   | **GPIO 11**  | **23**      | Такт SPI                    |
| **CS**       | **CE0**           | **GPIO 8**   | **24**      | `DISPLAY_SPI_DEVICE=0` → `spidev0.0` |
| **DC**       | Data/Command      | **GPIO 27**  | **13**      | `DISPLAY_GPIO_DC`           |
| **RST**      | Reset             | **GPIO 22**  | **15**      | `DISPLAY_GPIO_RST`          |
| **BL**       | Подсветка (GPIO)  | **GPIO 18*** | **12**      | `DISPLAY_GPIO_LIGHT` — см. ниже |

\*Если `DISPLAY_GPIO_LIGHT` в `.env` не задан, **luma** для `gpio_LIGHT` использует **BCM 18**. При конфликте с I2S/PCM задайте другой свободный пин и подключите **BL** к нему.

Полная карта GPIO: [pinout.xyz](https://pinout.xyz/).

## Переменные окружения в проекте

| Сигнал             | Переменная                         | По умолчанию                                | Назначение                    |
| ------------------ | ---------------------------------- | ------------------------------------------- | ----------------------------- |
| SPI MOSI / **DIN** | (шина SPI0)                        | **GPIO 10**                                 | Данные на контроллер          |
| SPI SCLK / **CLK** | (шина SPI0)                        | **GPIO 11**                                 | Такт SPI                      |
| **CS** / CE0       | `DISPLAY_SPI_DEVICE=0` + порт 0    | **GPIO 8**                                  | Выбор чипа, `spidev0.0`       |
| **DC**             | `DISPLAY_GPIO_DC`                  | **27**                                      | Команда / данные              |
| **RST**            | `DISPLAY_GPIO_RST`                 | **22**                                      | Сброс контроллера             |
| **BL**             | `DISPLAY_GPIO_LIGHT`               | не задано → в luma **18**                   | Управление подсветкой с GPIO  |
| Размер             | `DISPLAY_WIDTH` / `DISPLAY_HEIGHT` | 240×240                                     |                               |
| Скорость SPI       | `DISPLAY_SPI_SPEED_HZ`             | **40000000** (40 МГц)                       | Только дискретный ряд **luma**: 0.5–**52** МГц (шаги см. `display_agent.py`); значения вроде 80 МГц недопустимы |

На некоторых платах **DIN** подписан как SDA/MOSI/DI, **CLK** — SCL/SCK/SCLK — это те же линии, что в таблице выше.

## Подсветка: можно ли менять яркость?

### Как устроено железо

- Матрица **без подсветки** в обычных условиях **почти нечитаема** — нужна подсветка (LED за матрицей).
- На модуле обычно есть цепь **BL / LED / LED+**:
  - либо она заведена на **питание** (подсветка «всегда на» при поданном VCC — **нет** программного выключения с Pi);
  - либо на **отдельный вход**, который можно тянуть с GPIO (**вкл/выкл**) или через транзистор/драйвер — тогда софт может гасить подсветку.

### Что делает `pi_remote` / luma сейчас

- Драйвер **luma.lcd** `st7789` при создании устройства поднимает линию `gpio_LIGHT` (по умолчанию **BCM 18**), если подсветка к ней подключена — это по сути **включение подсветки** при старте агента.
- В текущем коде **pi_remote** нет настройки яркости с веб-интерфейса и **нет** ШИМ-диммирования: фактически доступно **включение при работе драйвера** и то, как luma держит пин при работе.
- **Плавная яркость (PWM)** зависит от того, выведен ли вход подсветки на пин с ШИМ, поддерживает ли обвязка модуля ток/частоту, и от возможностей библиотеки (для тонкой настройки смотрите [документацию luma.lcd](https://luma-lcd.readthedocs.io/) и разводку **вашей** платы).

### Итог

| Вопрос                                                      | Ответ                                                                                                                                                                                                                                       |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Обязательно ли подсвечивать?                                | Для нормального чтения экрана — **да** (светодиодная подсветка матрицы).                                                                                                                                                                    |
| Можно ли выключить программно?                              | **Да**, если линия **BL** подключена к GPIO и драйвер ею управляет; если BL сидит на **VCC** напрямую — **нет**, только обесточивать питание модуля или менять схему.                                                                       |
| Можно ли менять яркость из этого репозитория как настройку? | **Пока нет** отдельной настройки; возможно расширение (PWM / отдельный GPIO), если железо это позволяет.                                                                                                                                    |
| Конфликт пинов                                              | **GPIO 18** у luma по умолчанию совпадает с **PCM/I2S** на многих конфигурациях; при использовании I2S задайте другой пин через `DISPLAY_GPIO_LIGHT`. **BCM 13** в проекте занят **вентилятором** (`FAN_GPIO`) — не смешивать с подсветкой. |
