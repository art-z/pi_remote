# Дисплей ST7789 (SPI): распиновка и подсветка

Документ описывает подключение **как задумано в `pi_remote`** (переменные из `.env` / `.env.example`) и поведение **подсветки** с точки зрения софта и железа.

**Важно:** у разных модулей 240×240 на ST7789 **разная разводка разъёма**. Всегда сверяйтесь с **шелкографией вашей платы** (особенно **VCC 3.3 V vs 5 V**, **GND**, **BL / LED**). Ниже — согласование сигналов по **назначению**, а не по номеру пина на модуле.

## Сигналы и значения по умолчанию в проекте

Все номера GPIO — **BCM** (как в `raspi-config` и в коде luma).

| Сигнал | Переменная окружения | Значение по умолчанию | Назначение |
|--------|----------------------|------------------------|------------|
| SPI MOSI | (шина SPI0) | **GPIO 10** | Данные на контроллер |
| SPI SCLK | (шина SPI0) | **GPIO 11** | Такт SPI |
| CE0 / CS | `DISPLAY_SPI_DEVICE=0` + порт 0 | **GPIO 8** (CE0) | Выбор чипа на `spidev0.0` |
| D/C (Data/Command) | `DISPLAY_GPIO_DC` | **27** | Команда vs данные |
| RESET | `DISPLAY_GPIO_RST` | **22** | Сброс контроллера |
| Подсветка (BL) | `DISPLAY_GPIO_LIGHT` | не задано → в **luma** подставляется **18** | Вкл/выкл подсветки через GPIO (см. ниже) |
| Размер | `DISPLAY_WIDTH` / `DISPLAY_HEIGHT` | 240×240 | |
| Скорость SPI | `DISPLAY_SPI_SPEED_HZ` | 80000000 | Частота на шине (см. доку модуля) |

Если задать `DISPLAY_GPIO_LIGHT` явно, luma использует **указанный BCM-пин** вместо 18.

## Распиновка Raspberry Pi 40-pin (GPIO header)

Соответствие **BCM → физический номер пина** на стандартном заголовке J8 (Pi 4 / совместимые):

| BCM | Физ. pin | Функция в связке ST7789 SPI |
|-----|-----------|-----------------------------|
| 3V3 | 1 или 17 | Питание логики модуля (если модуль на 3.3 V) |
| 5V | 2 или 4 | Только если модуль рассчитан на 5 V питание — смотреть даташит |
| GND | 6, 9, 14, 20, … | Общий провод с модулем |
| **GPIO 8** | 24 | CE0 (CS) |
| **GPIO 10** | 19 | MOSI |
| **GPIO 11** | 23 | SCLK |
| **GPIO 18** | 12 | Подсветка по умолчанию (luma `gpio_LIGHT`), если BL выведен на GPIO |
| **GPIO 22** | 15 | RST (`DISPLAY_GPIO_RST`) |
| **GPIO 27** | 13 | D/C (`DISPLAY_GPIO_DC`) |

Полная карта пинов: [pinout.xyz](https://pinout.xyz/).

### Пример согласования с «типовой» маркировкой модуля

На многих платах встречаются имена вроде **VCC, GND, SCL/SCK, SDA/MOSI, RES/RST, DC, CS, BL**. Их нужно сопоставить строкам таблицы выше (не перепутать **MOSI** и **SCK**).

## Подсветка: можно ли менять яркость?

### Как устроено железо

- Матрица **без подсветки** в обычных условиях **почти нечитаема** — нужна подсветка (LED за матрицей).
- На модуле обычно есть цепь **BL / LED / LED+**:
  - либо она заведена на **питание** (подсветка «всегда на» при поданном VCC — **нет** программного выключения с Pi);
  - либо на **отдельный вход**, который можно тянуть с GPIO (**вкл/выкл**) или через транзистор/драйвер — тогда софт может гасить подсветку.

### Что делает `pi_remote` / luma сейчас

- Драйвер **luma.lcd** `st7789` при создании устройства поднимает линию **`gpio_LIGHT`** (по умолчанию **BCM 18**), если подсветка к ней подключена — это по сути **включение подсветки** при старте агента.
- В текущем коде **pi_remote нет** настройки яркости с веб-интерфейса и **нет** ШИМ-диммирования: фактически доступно **включение при работе драйвера** и то, как luma держит пин при работе.
- **Плавная яркость (PWM)** зависит от того, выведен ли вход подсветки на пин с ШИМ, поддерживает ли обвязка модуля ток/частоту, и от возможностей библиотеки (у luma обсуждались отдельные схемы с PWM; для тонкой настройки смотрите [документацию luma.lcd](https://luma-lcd.readthedocs.io/) и разводку **вашей** платы).

### Итог

| Вопрос | Ответ |
|--------|--------|
| Обязательно ли подсвечивать? | Для нормального чтения экрана — **да** (светодиодная подсветка матрицы). |
| Можно ли выключить программно? | **Да**, если линия **BL** подключена к GPIO и драйвер ею управляет; если BL сидит на **VCC** напрямую — **нет**, только обесточивать питание модуля или менять схему. |
| Можно ли менять яркость из этого репозитория как настройку? | **Пока нет** отдельной настройки; возможно расширение (PWM / отдельный GPIO), если железо это позволяет. |
| Конфликт пинов | **GPIO 18** у luma по умолчанию совпадает с **PCM/I2S** на многих конфигурациях; при использовании I2S задайте другой пин через `DISPLAY_GPIO_LIGHT`. **BCM 13** в проекте занят **вентилятором** (`FAN_GPIO`) — не смешивать с подсветкой. |
