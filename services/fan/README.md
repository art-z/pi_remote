# Вентилятор DC 5 V (PWM через GPIO)

Сервис **`fan`** крутит **4‑пиновый или 2‑пиновый DC‑вентилятор на 5 V** с управлением скоростью по температуре CPU: **ШИМ на BCM‑пине** (`fan_agent.py`, по умолчанию **GPIO 13**). Состояние пишется в Redis (`FAN_STATE_KEY`), API отдаёт его в `GET /api/status`.

**Важно:** с выхода GPIO **нельзя** напрямую питать вентилятор (ток и 3.3 V). Нужен **силовой ключ** на транзисторе (ниже).

## Схема подключения

Идея: **плюс вентилятора — на 5 V Raspberry Pi**, **минус — через низковольтный ключ** (транзистор) на общий **GND**. Управляющий сигнал **PWM** с Pi идёт **через резистор** на **затвор MOSFET** (или **базу NPN**).

### Цепь питания (два силовых провода вентилятора)

| Вывод вентилятора | Куда на Pi (J8) | Физ. pin |
| ----------------- | ----------------- | -------- |
| **+ (красный, VCC)** | **5 V** | **2** или **4** |
| **− (чёрный, GND)** | Не на GND Pi напрямую — на **сток n‑канального MOSFET** (или **коллектор NPN**) | — |

Другой вывод силового ключа: **исток MOSFET** / **эмиттер NPN** → **GND** Pi (например физ. **6**, **9**, **14**).

Так минус вентилятора оказывается «через транзистор» замкнутым на землю только когда ключ открыт — это и есть **ШИМ по оборотам**.

### Управление (GPIO → резистор → ключ)

| Сигнал | Куда | Примечание |
| ------ | ---- | ---------- |
| **BCM `FAN_GPIO`** (по умолчанию **13**, физ. pin **33**) | Через резистор на **затвор** (MOSFET) или **базу** (NPN) | Ограничить ток базы / демпфировать затвор |
| **GND** Pi | **Исток** / **эмиттер** и, при необходимости, **стяжка затвора** (например 10 кОм к GND) | Общая земля для логики и силы |

**Резистор:** типично **220 Ω–1 кОм** в цепи базы NPN (смотрите по выбранному транзистору и току коллектора); для логического MOSFET (например с низким порогом Vgs) часто **100 Ω–330 Ω** последовательно с затвором + **стяжка 10 кОм** затвор–GND, чтобы при старте Pi вентилятор не дёргался.

**Выбор ключа:** для 5 V и тока вентилятора (часто 0.1–0.3 А и выше) удобны **n‑MOSFET в режиме низковольтного ключа** (например IRLZ44N, AO3400 и аналоги по даташиту) или **NPN** (2N2222, BC337) при токе в пределах допустимого с учётом нагрева.

Не подключайте **минус вентилятора** напрямую к GPIO — только к **коллектору/стоку** ключа.

## Переменные окружения

См. `.env.example`:

| Переменная | По умолчанию | Назначение |
| ---------- | ------------- | ---------- |
| `FAN_GPIO` | `13` | BCM‑пин ШИМ |
| `FAN_PWM_HZ` | `25` | Частота PWM (Гц) |
| `FAN_TEMP_MIN` | `40` | Ниже — duty 0 % |
| `FAN_TEMP_MAX` | `65` | Выше — duty 100 % |
| `FAN_CHECK_INTERVAL_SEC` | `5` | Период опроса температуры |
| `FAN_STATE_KEY` | `fan:state` | Ключ Redis для JSON состояния |
| `CPU_TEMP_SYSFS` | (из API/общего `.env`) | Запасной путь температуры, если нет `vcgencmd` |

Температура в контейнере читается как в коде: сначала `vcgencmd measure_temp`, иначе sysfs.

## Docker

Сервис в `docker-compose.yml` с **`privileged: true`** и **`/dev/gpiomem`**, иначе `RPi.GPIO` не поднимет PWM.

## Конфликты с другими сервисами

В проекте **BCM 13** по умолчанию занят вентилятором. Подсветка дисплея и другие GPIO задавайте другими пинами (см. `services/display/README.md`).
